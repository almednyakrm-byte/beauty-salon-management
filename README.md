# نظام إدارة صالون تجميل (مواعيد، خدمات، دفع)
==========================

## ملخص المشروع
----------------

نظام إدارة صالون تجميل هو تطبيق إلكتروني يهدف إلى تسهيل إدارة المواعيد والخدمات والدفع في صالون تجميل. يتيح للمستخدمين إدارة المواعيد والخدمات والرسوم المالية بشكل مريح وسهل.

## بنية المشروع
----------------

### مجالات المشروع

* `app`: مجال تطبيق الويب
* `db`: مجال قاعدة البيانات
* `docker`: مجال docker

### مجالات التطبيق

* `models`: مجال النماذج (المعايير)
* `controllers`: مجال المديرين (الوظائف)
* `services`: مجال الخدمات (الوظائف المنطقية)
* `repositories`: مجال المستودعات (الوظائف التي تعمل مع قاعدة البيانات)

### قاعدة البيانات

* `users`: جدول المستخدمين
* `appointments`: جدول المواعيد
* `services`: جدول الخدمات
* `payments`: جدول الدفع

### أدوار المستخدمين

* `admin`: الدور الرئيسي (يمكنه إدارة كل شيء)
* `manager`: الدور الإداري (يمكنه إدارة المواعيد والخدمات)
* `user`: الدور المستخدم (يمكنه إدارة حسابه الخاص)

## إعداد البيئة باستخدام docker-compose
--------------------------------------

### خطوات الإعداد

1. تأكد من أن docker-compose مثبت على جهازك.
2. افتح مجلد المشروع في terminal.
3. استخدم الأمر التالي لتشغيل docker-compose:
bash
docker-compose up

4. افتح متصفح الويب وذهب إلى `http://localhost:8000` لفتح تطبيق الويب.

### تفاصيل docker-compose

* `docker-compose.yml`: الملف الرئيسي الذي يحتوي على تكوين docker-compose.
* `docker-compose.override.yml`: الملف الذي يحتوي على التكوين override.

* `services`:
 + `UserService`: الخدمة للمستخدمين
 + `AppointmentService`: الخدمة للمواعيد

 + `PaymentService`: الخدمة للدفع
* `repositories`:
 + `UserRepository`: المستودع للمستخدمين
 + `AppointmentRepository`: المستودع للمواعيد
 + `ServiceRepository`: المستودع للخدمات
 + `PaymentRepository`: المستودع للدفع

## تفاصيل قاعدة البيانات
------------------------

### جداول قاعدة البيانات

* `users`: جدول المستخدمين
 + `id`: العمود للرقم التعريفي
 + `name`: العمود للاسم
 + `email`: العمود للبريد الإلكتروني
 + `password`: العمود للكلمة السرية
* `appointments`: جدول المواعيد
 + `id`: العمود للرقم التعريفي
 + `user_id`: العمود للرقم التعريفي للمستخدم
 + `service_id`: العمود للرقم التعريفي للخدمة
 + `date`: العمود لل日期
 + `time`: العمود للوقت
* `services`: جدول الخدمات
 + `id`: العمود للرقم التعريفي
 + `name`: العمود للاسم
 + `price`: العمود للقيمة
* `payments`: جدول الدفع
 + `id`: العمود للرقم التعريفي
 + `user_id`: العمود للرقم التعريفي للمستخدم
 + `appointment_id`: العمود للرقم التعريفي للمواعيد
 + `amount`: العمود للقيمة

## تفاصيل أدوار المستخدمين
---------------------------

### أدوار المستخدمين

* `admin`: الدور الرئيسي (يمكنه إدارة كل شيء)
 + `create_user`: يمكنه إنشاء مستخدم جديد
 + `read_user`: يمكنه قراءة بيانات المستخدم
 + `update_user`: يمكنه تعديل بيانات المستخدم
 + `delete_user`: يمكنه حذف المستخدم
* `manager`: الدور الإداري (يمكنه إدارة المواعيد والخدمات)
 + `create_appointment`: يمكنه إنشاء موعد جديد
 + `read_appointment`: يمكنه قراءة بيانات المواعيد
 + `update_appointment`: يمكنه تعديل بيانات المواعيد
 + `delete_appointment`: يمكنه حذف المواعيد
 + `create_service`: يمكنه إنشاء خدمة جديدة
 + `read_service`: يمكنه قراءة بيانات الخدمات
 + `update_service`: يمكنه تعديل بيانات الخدمات
 + `delete_service`: يمكنه حذف الخدمات
* `user`: الدور المستخدم (يمكنه إدارة حسابه الخاص)
 + `read_user`: يمكنه قراءة بيانات المستخدم
 + `update_user`: يمكنه تعديل بيانات المستخدم
